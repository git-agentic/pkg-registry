# Sentinel v0.1.0-alpha.2

> ⚠️ **Alpha, pre-1.0.** Not production-ready; APIs may change without
> notice between prereleases.

A release-engineering iteration on
[v0.1.0-alpha.1](./v0.1.0-alpha.1.md) — **no functional changes to the
audit engine, proxy, sandbox, CLI, MCP server, steward, or action**. See the
alpha.1 notes for the full feature description; everything there applies
unchanged.

## Install

```bash
npm install -g @git-agentic/sentinel-cli@alpha @git-agentic/sentinel-proxy@alpha
```

## Changed since v0.1.0-alpha.1

- **npm trusted publishing (OIDC).** This release is published via GitHub
  Actions OIDC trusted publishing — no long-lived npm token exists anywhere
  in the pipeline. Provenance continues to be signed and recorded in the
  Sigstore transparency log for every package.
- **Release-pipeline hardening**, from the alpha.1 post-mortem: Stage B's
  tag verification survives being dispatched on the tag ref (forced
  re-fetch), and the publish loop passes unambiguous `./`-prefixed tarball
  paths so npm can never misparse them as git shorthands. Both failure modes
  stopped fail-closed before any registry contact in alpha.1's attempts.
- Version-carrying docs were genericized so version bumps no longer touch
  every README; the release smoke harness derives the expected version from
  the workspace itself.
